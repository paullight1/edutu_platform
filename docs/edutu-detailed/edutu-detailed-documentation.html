Architecture

Intended Boundary

Clients -> NestJS API -> Supabase

Actual implementation is transitional. Web, mobile, and admin still use Supabase directly for selected RLS reads/writes, auth, storage, and edge functions.

Identity

Clerk is primary for web/mobile. Backend accepts Clerk bearer tokens and Supabase fallback tokens. Standalone admin uses Supabase auth. Partner API uses API keys.

Backend

The active backend path is edutu-platform/backend/services/services/api. It boots with raw body support for Paystack webhooks, CORS allowlists, global Clerk/Supabase auth, throttling, scheduling, and feature modules.

Web and Admin

Standalone Admin

Supabase-authenticated admin dashboard with pages for dashboard, opportunities, users, creators, roadmaps, blog, settings, scraper, mobile control, and profile.

Web App

Clerk-authenticated Vite app with public marketing routes, `/app/*` product routes, premium gates, PWA, Capacitor Android, and embedded admin.

Waitlist

Next.js landing page with `/api/waitlist`, Neon Postgres, duplicate email handling, and SEO/OpenGraph metadata.

Mobile

Expo Router app under edutumobile. Root providers include Clerk, theme, offline, error boundary, deep-link listener, update prompt, mobile campaign host, and opportunity widget sync.

Core Services

@edutu/core owns opportunities, goals, chat, CV, payments, bookmarks, notifications, feature flags, analytics, deep links, hooks, and shared types.

Native Features

Push notifications, haptics, voice recording, text to speech, offline cache, native widgets, mobile campaigns, RevenueCat payments.

Data and Supabase

Data is split across Drizzle schema and multiple Supabase SQL homes. This is the highest structural risk in the repo.

Scraper and AI

Scraper Flow

Crawl4AI fetches source pages, BeautifulSoup extracts scholarship cards, DataCleaner normalizes and dedupes, Supabase upserts by canonical_url. NestJS scraper module handles admin runs, schedule, sources, jobs, and stats.

AI Control Plane

Routes include scraper extraction, opportunity enhancement/extraction/reranking, CV draft/tailor, roadmap questions/tags/match, quiz generation, chat coaching, and transcription.

Review Findings

  1. High Canonical opportunity schema drift across scraper, admin, Drizzle, and Supabase schemas.
  2. High User ID model mismatch between UUID-shaped backend IDs and mobile text Clerk IDs.
  3. High Marketplace financial operations need transaction/locking guarantees.
  4. High Possible missing-key auth weakness in n8n webhook.
  5. Review Scraper overlap and broad retention deletion risks.
  6. Review Mobile deep-link mismatch between parser and widget/campaign links.
  7. Review Paystack and RevenueCat entitlement source split.

Full findings are documented in 09-review-findings.md.

Task Backlog

The full backlog is in 08-task-backlog.md. Priority themes:

  • Make scraper imports locked, scoped, and observable.
